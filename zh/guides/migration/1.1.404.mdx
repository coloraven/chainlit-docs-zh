## 更新 Chainlit

通过将 Chainlit 更新到最新版本开始迁移：

```bash
pip install --upgrade chainlit
```

## 破坏性变更

### Python 版本要求

Chainlit 1.1.404 需要 Python 3.9 或更高版本。升级前请确保使用兼容的 Python 版本。

### 安全变更

Chainlit 现在监听 127.0.0.1（localhost）而非 0.0.0.0（公开），以提高安全性。

#### 容器化部署

若使用容器化部署，可能需要指定 `--host 0.0.0.0` 以使容器在新安全变更下正常工作。

## 新功能与变更

### 自定义配置位置的环境变量

现在可以使用 `CHAINLIT_APP_ROOT` 环境变量指定自定义配置位置。

### 改进的错误处理

- 数据层中的 HTTP 错误现在处理更优雅。
- 修复了 llama_index 集成中的 AttributeError。

### 配置更新

默认配置中 `edit_message` 的位置已修正。请检查 `config.toml` 文件并在必要时更新。

## 最佳实践

1. **检查 Python 环境**：确保使用 Python 3.9 或更高版本。
2. **更新容器化部署**：若使用容器，根据需要调整配置以包含 `--host 0.0.0.0`。
3. **检查自定义配置**：若自定义了 Chainlit 配置，请对照新默认值进行审查。
4. **测试集成**：若使用 llama_index 集成，升级后请彻底测试。

## 其他说明

- 修复了连接丢失后前端连接恢复的问题。
- 实现了基于 pytest 的新测试基础设施以提高稳定性。

升级后请彻底测试应用，以确保与这些变更兼容。
