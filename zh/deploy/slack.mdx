要让 Chainlit 应用在 Slack 上可用，你需要创建一个 Slack 应用并设置必要的环境变量。

## 工作原理

Slack 机器人会监听频道和私信中@它的消息。
它会根据上下文将回复发送到专用线程或私信。

| ✅      | ❌        | ✅       | ❌    | ❌       | ✅           | ❌            | ✅       |

## 安装 Slack Bolt 库

Slack Bolt 库不包含在 Chainlit 依赖中。你需要手动安装。

```bash
pip install slack_bolt
```

## 创建 Slack 应用

首先，导航到 [Slack apps dashboard for the Slack API](https://api.slack.com/apps)。在此你应该会看到一个 Create New App 绿色按钮。点击此按钮，选择从头创建应用的选项。

为机器人创建名称，例如 "ChainlitDemo"。选择你希望机器人所在的工作区。

<Frame caption="创建 Slack 应用">
  <img src="/images/slack/create-app.png" />
</Frame>

## 连接模式

Chainlit 支持两种连接 Slack 的方式：

- **HTTP 模式**（默认）：Slack 通过 HTTP 向 Chainlit 服务器发送事件。需要公共 URL。
- **Socket 模式**（自 2.7.0 起）：Chainlit 通过 WebSocket 连接 Slack。无需公共 URL — 非常适合本地开发或受限网络。

要使用 Socket 模式，请设置 `SLACK_WEBSOCKET_TOKEN` 环境变量（参见[环境变量](#set-the-environment-variables)）。设置后，Socket 模式优先于 HTTP 处理器。

## 本地开发

### 使用 Socket 模式（推荐）

如果使用 Socket 模式，无需公共 URL 或 ngrok。Chainlit 直接通过 WebSocket 连接 Slack。参见下方的 [Socket 模式应用清单](#socket-mode)。

### 使用 HTTP 模式

如果在本地使用 HTTP 模式，必须将本地 Chainlit 应用暴露到互联网，以接收来自 Slack 的传入消息。你可以使用 [ngrok](https://ngrok.com/)。

```bash
ngrok http 8000
```

这将给你一个公共 URL，可用于设置应用清单。部署 Chainlit 到公共主机后，别忘了替换它。

## 设置应用清单

前往 App Manifest，粘贴以下 Yaml。

<Note>将 `{placeholders}` 替换为你自己的值。</Note>

```yaml
display_information:
  name: { APP_NAME }
features:
  bot_user:
    display_name: { APP_NAME }
    always_online: false
oauth_config:
  scopes:
    user:
      - im:history
      - channels:history
    bot:
      - app_mentions:read
      - channels:read
      - chat:write
      - files:read
      - files:write
      - im:history
      - im:read
      - im:write
      - users:read
      - users:read.email
      - channels:history
      - groups:history
settings:
  event_subscriptions:
    request_url: https://{ CHAINLIT_APP_HOST }/slack/events
    bot_events:
      - app_home_opened
      - app_mention
      - message.im
  interactivity:
    is_enabled: true
    request_url: https://{ CHAINLIT_APP_HOST }/slack/events
  org_deploy_enabled: false
  socket_mode_enabled: false
  token_rotation_enabled: false
```

点击 Save Changes。

<Frame caption="设置应用清单">
  <img src="/images/slack/app-manifest.png" />
</Frame>

你会看到 URL 未验证的警告。暂时可以忽略。

### Socket 模式

如果使用 Socket 模式，对上面的清单进行以下更改：删除两个 `request_url` 字段，并将 `socket_mode_enabled` 设置为 `true`。

```yaml
settings:
  event_subscriptions:
    request_url: https://{ CHAINLIT_APP_HOST }/slack/events # [!code --]
    bot_events:
      - app_home_opened
      - app_mention
      - message.im
  interactivity:
    is_enabled: true
    request_url: https://{ CHAINLIT_APP_HOST }/slack/events # [!code --]
  org_deploy_enabled: false
  socket_mode_enabled: false # [!code --]
  socket_mode_enabled: true # [!code ++]
  token_rotation_enabled: false
```

## [可选] 收到消息时的表情回应

添加可选功能，在收到 Slack 消息时显示表情回应，在机器人处理请求时为提供即时用户反馈。

<Note>
  此功能需要 `reactions:write` OAuth scope。如果启用此功能，你需要将此 scope 添加到 Slack 应用的 OAuth 配置中。
</Note>

要启用此功能，在 `chainlit.md` 文件中添加以下配置：

```toml
[features.slack]
reaction_on_message_received = true
```

<Warning>
  此功能默认禁用以保持向后兼容。如果启用此功能，你需要将 `reactions:write` scope 添加到 Slack 应用的 OAuth 配置中。
</Warning>

## [可选] 处理用户的表情回应

响应用户在 Slack 工作区中为消息添加的表情回应。

<Note>
  此功能需要 `reactions:read` OAuth scope 和 `reaction_added` bot 事件。将两者都添加到 Slack 应用的清单和 OAuth 配置中。
</Note>

```python
from typing import Any, Dict
import chainlit as cl

@cl.on_slack_reaction_added
async def handle_reaction(event: Dict[str, Any]):
    reaction = event.get("reaction")       # 例如 "thumbsup"
    user_id = event.get("user")            # Slack 用户 ID
    item = event.get("item", {})           # 被回应项信息

    print(f"用户 {user_id} 在频道 {item.get('channel')} 中用 :{reaction}: 回应")
```

`event` 字典包含：

| 键 | 描述 |
| --- | ----------- |
| `reaction` | 不带冒号的表情名称（例如 `"thumbsup"`） |
| `user` | 添加回应的 Slack 用户 ID |
| `item` | 包含被回应项的 `type`、`ts`（时间戳）和 `channel` 的字典 |

<Note>
  自版本 **2.8.5** 起。
</Note>

## 将 Slack 应用安装到工作区

导航到 Install App 标签页，点击 Install to Workspace。

## 设置环境变量

<Note>在应用代码之外设置环境变量。</Note>

### Bot Token

安装 Slack 应用后，你会看到 Bot User OAuth Token。将其在 Chainlit 应用中设置为环境变量。

<Frame caption="复制 Bot Token">
  <img src="/images/slack/bot-token.png" />
</Frame>

```bash
SLACK_BOT_TOKEN=your_bot_token
```

### Signing Secret

导航到 Basic Information 标签页，复制 Signing Secret。然后在 Chainlit 应用中将其设置为环境变量。

<Frame caption="复制 Signing Secret">
  <img src="/images/slack/signing-secret.png" />
</Frame>

```bash
SLACK_SIGNING_SECRET=your_signing_secret
```

### App-Level Token（仅 Socket 模式）

如果使用 Socket 模式，还需要 App-Level Token。导航到 **Basic Information** > **App-Level Tokens**，创建具有 `connections:write` scope 的令牌，并将其设置为环境变量。

```bash
SLACK_WEBSOCKET_TOKEN=your_app_level_token
```

<Note>
  设置 `SLACK_WEBSOCKET_TOKEN` 后，Chainlit 使用 Socket 模式，HTTP `/slack/events` 端点不会注册。在 Socket 模式下不需要 `SLACK_SIGNING_SECRET`。
</Note>

## 启动 Chainlit 应用

由于 Chainlit 应用未运行，Slack 应用将无法与其通信。

示例中，我们将使用这个简单应用：

```python my_app.py
import chainlit as cl

@cl.on_message
async def on_message(msg: cl.Message):
    # 访问原始 slack 事件
    print(cl.user_session.get("slack_event"))
    # 访问 slack 用户
    print(cl.user_session.get("user"))

    # 访问可能附加的文件
    attached_files = msg.elements

    await cl.Message(content="你好，世界").send()
```

<Note>
  提醒：确保环境变量已设置。如果使用 HTTP 模式，
  本地 Chainlit 应用必须通过 ngrok 暴露到互联网。
</Note>

启动 Chainlit 应用：

```bash
chainlit run my_app.py -h
```

<Note>
  使用 -h 以避免打开默认 Chainlit UI，因为我们使用的是 Slack。
</Note>

你现在应该可以通过 Slack 与 Chainlit 应用交互了。

## 聊天历史

聊天历史可直接通过 `fetch_slack_message_history` 方法获取。
它将获取当前线程或私信频道中的最近消息。

```python
import chainlit as cl
import discord


@cl.on_message
async def on_message(msg: cl.Message):
    fetch_slack_message_history = cl.user_session.get("fetch_slack_message_history")

    if fetch_slack_message_history:
        print(await fetch_slack_message_history(limit=10))

    # 你的代码写在这里
```
