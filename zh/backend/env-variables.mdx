---
title: 环境变量
---

在代码中硬编码 API 密钥不是好的做法。这会使代码可移植性和灵活性降低，也更难保证代码安全。相反，应使用环境变量存储特定于开发环境的值。

Chainlit 会自动从项目根目录的 `.env` 文件加载环境变量。此文件应添加到 `.gitignore` 中，以免提交到仓库。

```bash .env
OPENAI_API_KEY=sk-...
PINECONE_API_KEY=...
```

## 公开应用与环境变量

若要将应用分享给更广泛的受众，不应在 `.env` 文件中放置自己的 OpenAI API 密钥。
相反，应在 Chainlit 配置中使用 `user_env`，让每位用户提供自己的密钥。

然后可以在代码中通过以下方式访问用户的密钥：

```python
import chainlit as cl

user_env = cl.user_session.get("env")
```
